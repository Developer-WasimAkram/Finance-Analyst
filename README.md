# 🏠 Seattle Listings - CSV to PostgreSQL Data Loader

## What Does This Project Do?

Imagine you have a big notebook (CSV file) full of information about houses and rooms for rent in Seattle. This project does two things:

1. **Takes that notebook and saves it into a super organized digital filing cabinet (PostgreSQL database)** — so it's easy to search and find things later.
2. **Takes the data back out of the filing cabinet and saves it into a new notebook (CSV file)** — so you can share it or look at it in Excel.

Think of it like this:
```
📓 CSV File (your notebook)
    ⬇️  python_to_sql_data_loader.py sends it DOWN into the database
💾 PostgreSQL Database (the filing cabinet)
    ⬇️  sql_to_python_data_loader.py pulls it back OUT
📓 New CSV File (a fresh copy of your notebook)
```

## What's in the Data?

The `listings.csv` file has information about rental listings in Seattle. Each row is one listing, and the columns are:

| Column | What It Means |
|--------|--------------|
| `id` | A unique number for each listing (like a student ID) |
| `name` | The name of the listing (e.g., "Fab, private seattle urban cottage!") |
| `host_id` | A unique number for the person renting it out |
| `host_name` | The name of the person renting it out |
| `neighbourhood_group` | The big area in Seattle (like "West Seattle") |
| `neighbourhood` | The smaller neighborhood (like "Wallingford") |
| `latitude` | How far north or south the place is on a map |
| `longitude` | How far east or west the place is on a map |
| `room_type` | What kind of space it is (whole home, private room, etc.) |
| `price` | How much it costs per night in dollars |
| `minimum_nights` | The minimum number of nights you have to stay |
| `number_of_reviews` | How many people left a review |
| `last_review` | The date of the last review |
| `reviews_per_month` | How many reviews come in each month |
| `calculated_host_listings_count` | How many listings this host has |
| `availability_365` | How many days per year the listing is available |
| `number_of_reviews_ltm` | Number of reviews in the last 12 months |
| `license` | The license number for the listing |

## Before You Start (Things You Need)

1. **Python** — The programming language that runs the code (version 3.x)
2. **PostgreSQL** — The database software, installed and running on your computer
   - Username: `postgres`
   - Password: `admin`
   - Port: `5432`
3. **Python Libraries** — Install them by running this command in your terminal:

```bash
pip install -r requirements.txt
```

This installs:
- **pandas** — Helps Python read and work with tables of data (like a spreadsheet helper)
- **sqlalchemy** — Helps Python talk to databases (like a translator between Python and PostgreSQL)
- **psycopg2-binary** — The driver that actually connects Python to PostgreSQL (like a USB cable between two devices)

## How to Run the Code (Step by Step)

### Step 1: Load CSV Data INTO the Database

Run this command in your terminal:

```bash
python python_to_sql_data_loader.py
```

**What happens behind the scenes (line by line):**

```python
import os
import pandas as pd
# ← Bring in the os and pandas tools.

from sqlalchemy import create_engine
# ← Bring in the create_engine function from sqlalchemy.

BASE_DIR = os.path.dirname(os.path.abspath(__file__))
# ← Get the folder where this script lives. Works on any computer!

df = pd.read_csv(os.path.join(BASE_DIR, 'data', 'listings.csv'))
# ← Read the CSV file using a relative path.
#   os.path.join() builds the correct path for any OS (Windows, Mac, Linux).

engine = create_engine('postgresql://postgres:admin@localhost:5432/postgres')
# ← Create a connection to the PostgreSQL database.
#   Let's break down the address:
#     postgres   = username (who you are)
#     admin      = password (your secret key)
#     localhost  = the computer (your own computer)
#     5432       = the port number (which door to knock on)
#     postgres   = the database name (which filing cabinet to open)

df.to_sql('listings', engine, if_exists='replace', index=False)
# ← Take the spreadsheet "df" and save it into the database as a table called "listings".
#   if_exists='replace' = if the table already exists, delete the old one and make a new one
#   index=False         = don't add an extra number column (we don't need it)
```

**Result:** All 6,442 rows from the CSV are now stored in the PostgreSQL database!

---

### Step 2: Pull Data FROM the Database and Save as CSV

Run this command in your terminal:

```bash
python sql_to_python_data_loader.py
```

**What happens behind the scenes (line by line):**

```python
import os
import pandas as pd
# ← Bring in the os and pandas tools.

from sqlalchemy import create_engine
# ← Bring in the create_engine function again.

BASE_DIR = os.path.dirname(os.path.abspath(__file__))
# ← Get the folder where this script lives.

engine = create_engine('postgresql://postgres:admin@localhost:5432/postgres')
# ← Create a connection to the database (same address as before).

df = pd.read_sql('SELECT * FROM listings', engine)
# ← Ask the database a question using SQL language:
#   "SELECT *" means "give me ALL columns"
#   "FROM listings" means "from the table called listings"
#   The answer gets stored in "df" — our spreadsheet in Python.

df.to_csv(os.path.join(BASE_DIR, 'data', 'from_sql.csv'), index=False)
# ← Save the spreadsheet as a new CSV file using a relative path.
#   Works on any computer!

print(f'Saved {len(df)} rows to data/from_sql.csv')
# ← Print a message on the screen telling us how many rows were saved.
#   len(df) counts the number of rows. It will say "Saved 6442 rows to data/from_sql.csv"
```

**Result:** You now have a fresh CSV file at `data/from_sql.csv` with all the data from the database!

### Want Different Data? Change the SQL Query!

In `sql_to_python_data_loader.py`, you can change the `SELECT * FROM listings` part to pull only the data you need. Think of SQL like asking a librarian a question — you can be very specific about what you want!

Here are some examples you can try by replacing the query string:

#### Get only cheap listings (price under $50)
```python
df = pd.read_sql('SELECT * FROM listings WHERE price < 50', engine)
```
> "Hey database, give me only the listings that cost less than $50 per night."

#### Get only entire homes (no shared rooms)
```python
df = pd.read_sql("SELECT * FROM listings WHERE room_type = 'Entire home/apt'", engine)
```
> "Hey database, give me only the listings where you get the whole place to yourself."

#### Get listings in a specific neighborhood
```python
df = pd.read_sql("SELECT * FROM listings WHERE neighbourhood = 'Wallingford'", engine)
```
> "Hey database, give me only the listings in the Wallingford neighborhood."

#### Get only the name, price, and neighborhood columns
```python
df = pd.read_sql('SELECT name, price, neighbourhood FROM listings', engine)
```
> "Hey database, I don't need all 18 columns — just give me the name, price, and neighborhood."

#### Get the top 10 most reviewed listings
```python
df = pd.read_sql('SELECT * FROM listings ORDER BY number_of_reviews DESC LIMIT 10', engine)
```
> "Hey database, sort all listings by number of reviews (most first), then give me only the top 10."

#### Get the average price by room type
```python
df = pd.read_sql('SELECT room_type, AVG(price) as avg_price FROM listings GROUP BY room_type', engine)
```
> "Hey database, group the listings by room type and tell me the average price for each group."

#### Combine multiple conditions (cheap + good reviews + entire home)
```python
df = pd.read_sql("""
    SELECT name, price, neighbourhood, number_of_reviews 
    FROM listings 
    WHERE price < 100 
    AND number_of_reviews > 50 
    AND room_type = 'Entire home/apt'
    ORDER BY price
""", engine)
```
> "Hey database, find me entire homes under $100/night with more than 50 reviews, sorted cheapest first."
